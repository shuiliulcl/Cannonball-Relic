# 咒语栏显示标准

本文档定义幸存者模式底部咒语栏的显示与交互标准。后续新增咒语、技能碎片、隐藏 combo 或阶段式技能时，都按此标准实现和验收。

## 核心原则

1. 只展示真实可触发按键。
   按钮左侧只能出现 `Q`、`E`、`R`、`1` 到 `9` 这类实际快捷键。不要把图标、符号、装饰 glyph 放在按键位置，避免玩家误以为它们也是输入键。

2. 阶段式技能必须成组展示。
   如果多个按钮属于同一个技能链，不能平铺成互不相关的技能。必须用一个视觉分组包起来，并在组标题里说明它们是同一技能链。

3. 咒语信息必须无需拉动横向滑条即可看全。
   底部咒语栏必须自动换行。任何屏幕宽度下，都不应要求玩家拖动横向滑条才能看到后面的咒语按钮。

4. 不可点击状态要真实。
   升级选 Buff、开局遮罩、结算遮罩出现时，底部咒语栏必须隐藏。不要留下看似可点但实际被遮罩拦截的按钮。

5. 点击后状态必须立即反馈。
   消耗声能、充能、瞄准、发射、进入冷却或失败提示后，按钮状态需要立刻刷新，不能等下一轮自然渲染才变化。

## 标准结构

底部咒语栏由两层组成：

- `survivor-command-dock`：底部容器，负责定位、显隐和整体布局。
- `survivor-command-list`：按钮列表，必须允许换行，禁止横向滚动。

阶段式技能需要额外分组：

- `survivor-cannon-chain`：阶段式技能容器。
- `survivor-cannon-chain-title`：技能链标题和说明。
- `survivor-cannon-chain-actions`：阶段按钮区域。

当前“人间大炮”标准展示为：

- 组标题：`人间大炮`
- 组说明：`同一技能链：准备 / 瞄准 / 发射`
- `Q 一级准备`：消耗声能并增加大炮充能。
- `E 瞄准`：锁定敌群或进入鼠标微调。
- `R 发射`：消耗大炮槽并发射玩家。

## 按钮内容

每个咒语按钮只保留三类信息：

- 快捷键：真实输入键，位于最左侧。
- 咒语名称与消耗/条件：例如 `一级准备 / 34 声能`、`瞄准 / 1/3 准备`。
- 状态徽标：例如 `充能`、`待装填`、`待瞄准`、`发射`、`声能不足`。

按钮内不放装饰图标。若未来需要视觉识别，应通过边框、颜色、背景、分组标题或状态徽标表达，不占用快捷键位置。

## 布局规则

1. `survivor-command-list` 使用 `flex-wrap: wrap`。
2. `survivor-command-list` 不使用 `overflow-x: auto`，也不显示横向 scrollbar。
3. 单个普通咒语按钮宽度可以固定或响应式收缩，但不能挤压到文字重叠。
4. 技能链在桌面端可以占据一段固定宽度，在移动端必须占满一行，内部阶段按钮等分排布。
5. 底栏高度变化后，资源面板和角色数值面板必须自动上移，避免覆盖。

## 响应式验收

新增或调整咒语栏时，至少验证以下场景：

1. 桌面默认窗口：开局后 Q/E/R 可见，且“人间大炮”是一个技能链分组。
2. 移动端 `390x700`：Q/E/R 三个阶段按钮完整可见，没有横向页面滚动。
3. 解锁大量咒语：使用 GM 的 `All VFX` 后，所有底部按钮自动换行展示，`scrollWidth` 不大于 `clientWidth`。
4. 升级弹窗：底部咒语栏隐藏，不出现“看得到但点不到”的按钮。
5. 状态流转：按 Q 后 E 立即可用；按 E 后 R 立即可用；失败状态也要立即刷新提示。

## 新增咒语要求

普通单段咒语：

- 使用数字键从 `1` 开始顺序分配。
- 加入 `survivor-command-list`。
- 不需要独立分组，除非它属于组合链或阶段链。

阶段式或 combo 链咒语：

- 必须有一个明确的组标题。
- 组内按钮要用动词命名阶段，例如 `准备`、`瞄准`、`发射`、`锁定`、`引爆`。
- 每个阶段仍需绑定真实快捷键。
- 效果说明要放在状态、meta 或组说明里，不通过额外装饰符号表达。

隐藏 combo：

- 未触发前不强制占用底栏。
- 如果触发后进入可手动释放状态，按普通咒语或阶段链规则加入底栏。

## 禁止项

- 禁止用横向滑条作为查看完整咒语列表的主要方式。
- 禁止把装饰符号放在快捷键旁边，让它看起来像另一个可按键。
- 禁止把同一技能的多个阶段展示成彼此独立的普通技能。
- 禁止弹窗期间保留底部按钮的可见状态。
- 禁止新增按钮后不做移动端检查。
