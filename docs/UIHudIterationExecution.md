# UI HUD Iteration Execution

更新日期：2026-05-25

## 背景

当前战斗 HUD 的主要问题是底部常驻信息面板遮挡战斗画面，玩家在靠近屏幕下方时容易看不到主角、怪物或弹珠路径；同时角色血量不够醒目，玩家需要额外寻找关键信息，影响走位和弹珠回收判断。

本次迭代目标是在不改动战斗逻辑的前提下，先完成一版更适合实时战斗的 HUD 布局：关键信息靠屏幕上缘和侧边呈现，底部战斗区域保持干净。

## 设计目标

- 血量必须一眼可读，固定在左上角，使用大字号数值和高对比血条。
- 底部不放常驻大面板，避免遮挡主角、怪物、弹珠或弹道。
- 弹珠状态、蓄力、增伤倍率、闪避冷却集中在右上战斗状态区。
- 操作说明保留在左侧中段，明确展示 `WASD`、`左键`、`右键`、`Space` 对应功能。
- 窄窗口下也要显示操作说明，且不回到底部遮挡视野。

## 视觉方案

设计图：`docs/ui-hud-iteration-v1.png`

核心布局：

- 左上：`HP`、当前/最大血量、血条、护盾状态。
- 顶部中间：当前波次和战斗进度条。
- 右上：战斗状态面板，包含蓄力、弹珠状态、反弹增伤倍率、闪避冷却、分数。
- 右上角：强化列表入口按钮。
- 左侧中段：轻量操作说明条。

## 实施内容

- 重构 `index.html` 中 HUD DOM 结构，移除底部常驻强化/操作大面板。
- 调整 `src/styles.css` 中 HUD 视觉样式，从厚重卡片式面板改为深色半透明战斗 HUD。
- 更新 `src/ui/Hud.ts` 中状态 class 绑定：
  - 低血量状态：`hp-panel.is-low`
  - 护盾空状态：`shields-badge.is-empty`
  - 闪避就绪状态：`dash-meter.is-ready`
  - 增伤高亮状态：`damage-scale.is-boosted`
  - 蓄力环角度：`--charge-angle`
- 保留强化列表按钮交互和原有 HUD 数据绑定 ID，降低接入风险。

## 验收标准

- `npm run build` 通过。
- 开局进入战斗后，底部没有常驻 HUD 面板遮挡。
- 血量与护盾在左上角清晰可读。
- 操作说明可见，并明确展示移动、发射、回收、闪避按键。
- 窄窗口下操作说明仍可见。
- 浏览器检查无 console error。

## 合并说明

合并远端提交 `486ad6c Refine combat HUD layout` 时，远端版本包含一版 HUD 调整，但其中蓄力/状态区域仍存在底部常驻面板思路。最终以本次已体验确认的方案为准：保留远端提交历史，在其基础上解决冲突并维持底部无遮挡布局。

## 后续建议

- 将弹珠回收倒计时改为跟随弹珠或弹道附近的上下文标签。
- 将闪避冷却进一步做成贴近角色的环形读条，减少玩家视线跳转。
- 增加低血量短促闪烁或边缘警示，但需要避免持续干扰瞄准。
- 根据真实分辨率和关卡镜头，补一轮 16:9、窄屏、低高度窗口的视觉验收截图。
